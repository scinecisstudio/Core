Privacy Policy

Last updated: February 28, 2026

This policy applies to the Core Android app (com.scinecis.launcher).

1. Information We Process

Depending on features you enable, Core may process:

  • Installed app list and package names (launcher functionality and app organization).
  • Contacts data (for contact search and Quick Tile favorites).
  • Calendar event data (for "Today" events and countdown display).
  • Notification metadata from Android notification listener (for badge counts).
  • Accessibility event metadata (for gesture features and app lock behavior).
  • User-entered weather location and weather API key.
  • App usage/configuration event metadata for analytics and crash diagnostics.

2. Data Stored Locally on Device

Core stores launcher settings and related records locally, including preferences and local database entries (for example app status, selected contacts, custom categories, and shortcuts).

Core also allows optional export/import of backup files that you control.

3. Third-Party Services

Core uses third-party processors to operate certain features:

  • Google Firebase Analytics
  • Google Firebase Crashlytics
  • Google Firebase Performance Monitoring
  • Google Firebase Remote Config
  • OpenWeather APIs

These services may receive technical and operational metadata needed for app analytics, diagnostics, feature configuration, and weather responses.

Firebase: Privacy
OpenWeather: Privacy Policy

4. Why We Process Data

  • Provide core launcher capabilities (app listing, search, categories, widgets).
  • Enable optional user-requested features (contacts, calendar, weather, badges, app lock).
  • Measure stability/performance and diagnose crashes or reliability issues.
  • Apply remote configuration values for supported app behavior.

5. Data Sharing and Sale

We do not sell personal information.

We share data only with service providers required to run app features and diagnostics described in this policy.

6. Permissions and Feature Mapping

  • READ_CONTACTS: contact search and Quick Tile contact selection.
  • READ_CALENDAR: display and countdown of calendar events.
  • CALL_PHONE: direct call action from contact cards.
  • SYSTEM_ALERT_WINDOW: optional internet speed overlay.
  • QUERY_ALL_PACKAGES: launcher app discovery and app management features.
  • Accessibility Service: gesture shortcuts and app lock flow support.
  • Notification Listener: app icon notification badge counts.

If a permission is denied or revoked, related features may not function.

7. Retention

  • Local data remains on your device until you delete it, clear app data, or uninstall the app.
  • Exported backup files remain where you save them until you delete them.
  • Third-party data retention is controlled by each provider's policy.

8. Your Choices and Controls

  • Grant, deny, or revoke permissions in Android settings at any time.
  • Disable optional features inside app settings.
  • Clear app data or uninstall the app to remove local data.

9. Security

We use reasonable safeguards to protect information, but no method of transmission or storage is fully secure.

10. Children's Privacy

Core is not directed to children under 13. If you believe a child has provided personal data through the app, contact us so we can address it.

11. Play Store Data Safety Alignment

Your Google Play Data Safety form should match this policy, including all enabled SDKs, permissions, and data processing behavior in each release.

12. Changes to This Policy

We may update this policy over time. Material updates will be reflected by changing the "Last updated" date on this page.
